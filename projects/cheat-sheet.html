Ruby Enumerables

e-nu-mer-a-ble
adjective
  1. countable
  2. able to be counted by one-to-one correspondence with the set of all positive integers.

Enumerable methods are ones which are common to all collections in Ruby. Each data structure (sets, arrays, hashes, etc.) still have their own specific methods, but enumerables work on any of them that have a defined method 'each.'

Here are some useful enumerable methods to remember:

.all? -

Passes each element of the collection to the given block. The method returns true if the block never returns false or nil. If the block is not given, Ruby adds a block of { |obj| obj } which will cause all? to return true when none of the collection members are false or nil.

[nil, true, 99].all? = false

[1, 2, 3].all? {|i| i < 4} = true

.each_with_index -

Runs block over two arguments, one for the item in the array, and one for the index.

["foo", "bar"].each_with_index {|item, index| puts "#{item} is at index #{index}"}

"foo is at index 0"
"bar is at index 1"

.map -

Runs block over collection and outputs a new collection with the result.

array = [1, 2, 3]

new_array = array.map {|i| i + 1}
p new_array
[2, 3, 4]
