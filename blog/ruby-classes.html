Your Own Virtual Assembly Line

October 18, 2015

Hey gang, today we're talking about something fun and interesting: Classes.

First things first, what is a class? A class is simply an object in Ruby that acts as a template or blueprint to create other objects. That may seem a bit confusing, but it becomes easier to understand if you think about real-world object groups and what defines them. For instance, all TVs have a screen, power button, channel change, and volume adjustment. All of those things, especially the screen, are what define a television. Knowing that, we can easily make a program that replicates the things that we know are always true for a TV. Take a look at this example:

Well there we have it. That's a full class. Unfortunately, it doesn't do a thing for us at the moment. We have to tell it that we want it to act like a TV when we start it! To do that, we need what's known as an initialize method. Basically this does what it sounds like: it will set certain things about the object we're making to be true when we create a new one. Let's take a look:

That's a bit better! What you see here is us telling the computer that when you create a new Television object, you specify the screen size, which gets saved in the instance variable for @screen. It also tells the computer that when we create a new Television object, the power will be off, the volume will be at 0, and the channel will start at 3. But this still doesn't do a whole lot for us yet. Those words with the little @ symbols? Those guys are called instance variables. This is one of the things that allows classes to function in the first place. A variable, unless otherwise specified, is typically a local variable. This means it can't be carried from inside a method to outside one, or vice versa. An instance variable however, allows us to pass values across multiple methods. Cool, right? So you can see from this example, the when we create a new Television object from this class, it'll start with certain attributes.

Reviewing what I talked about in the first paragraph, what this program is doing, is allowing us to easily create a TV object which we then know has certain properties upon creation, and allows us to manipulate it in repeatable ways. Classes are powerful tools in Ruby, and you can create all sorts of interesting things using them.
