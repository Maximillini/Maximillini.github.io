Les Enumerables

October 11, 2015

Wow, what a week. Methods, methods, methods!

This week, I'm going to be talking about a really interesting and powerful tool called an Enumberable method. Specifically, I'm going to be explaining the Enumberable#map method.

Before I can explain what .map will do for you, I'll have to explain what iteration is and what it does for collection objects.

Collection objects are simply objects that contain collections or lists of other things. These would be objects such as arrays, hashes, sets, and ranges. With iteration, we can run code over (or iterate over) each element of the collection object. This in itself is an immensely powerful feature of programming in general, allowing us efficiency and letting the computer do all the heavy lifting. Let's take a look at an example:

def add_one(array)
array.each { |i| i + 1 }
end

What this is saying is: "for every element that's in the array, add one to it." It also works by taking one element at a time, running the code in the curly braces, and spitting it back out.

From there, we can drill one level deeper on what an Enumerable method is. An enumerable method is one that is shared across multiple collection objects. Technically, Enumerable is a module included in all collection objects by default, and it allows for the use of iteration methods, specifically .each. In fact, one of the requirements for Enumerable to be included in a class, is for the .each method to be defined!

Now that you understand a little better about what iteration and what Enumerables are, it becomes much easier to grasp what the Enumerable method .map will do for us. The .each method and .map methods are actually very similar, so understanding .map will just be adding to what we already know about .each. Where .each will iterate over an array or hash and return the values of our array or hash with the new results, .map will collect those results into a brand new array object! This allows us to create a non-destructive method for changing our collection object and RETAINING those changes in a new array. Amazing, right? .map's syntax works the same way as .each, but let's look at an example:

array = [1, 2, 3]
array.each { |i| i + 1 }
y = array.map { |i| i * 1 }

In this example, .each will return the output of the block of code as performed on each element of the array, but the original array elements are untouched. However, with .map, we've actually created an entirely new array, which we've assigned to y. Now that you understand these concepts, go forth and code!
